# 의료 콘텐츠 팩트체크 체크리스트

태림한의원 모든 의료 글 발행 전 적용. CI(`medical-content-lint.mjs`)가 **표현**은 자동 차단하지만, **사실 정확성**은 사람이 확인해야 한다.

## 자동 (CI에서 차단)
- 효과 단정·과장·최상급 표현 (완치, 100%, 가장 효과적, 부작용 없음 등)
- 출처(citations) 없는 발행 글
- em dash

실행: `node scripts/medical-content-lint.mjs --strict`

## 수동 (사람이 발행 전 확인)

### 1. 사실 정확성 (가장 중요)
- [ ] **비용·수가**: 건강보험심사평가원·보건복지부 1차 출처로 확인 (매년 변동, 추측 금지)
- [ ] **통계 수치**(%, 발생률, 유병률): 1차 출처 확인 또는 "보고된 바로는" 톤
- [ ] **제도·법**(보험 적용, 횟수 한도): 정부 고시 직접 확인
- [ ] **효과**: 학술 논문(PubMed/학회지)으로 근거 확인

### 2. 출처 (citations)
- [ ] citations 최소 1개 (1차 출처)
- [ ] 효과를 언급하는 글: 학술 논문 citation 포함 (정부·학회 사이트만으로 부족)
- [ ] **AI 환각 금지**: 논문 인용 시 실재 여부 웹검색으로 검증 (가짜 저자·저널·DOI 절대 금지)
- [ ] url 실제 접속 확인 (깨진 링크 X)

### 3. 표현 (CI + 재확인)
- [ ] 효과는 "도움이 됩니다", "보고되었습니다", "개선에 기여" 톤
- [ ] 단정·보장("낫습니다", "확실히") 회피
- [ ] em dash 없음 (쉼표·하이픈)

### 4. 명의 (의료법 27조·56조)
- [ ] author = 한의사 slug (seo-johyuk)
- [ ] "편집팀"·"에디터" 등 무자격 1인칭 표현 없음
- [ ] 본문 1인칭: "원장이", "진료실에서"

## 발행 흐름
1. 글 작성
2. `node scripts/medical-content-lint.mjs --strict` 통과 (표현·출처)
3. 위 수동 체크리스트 (사실 정확성)
4. `git push` → CI 재검사 → 배포 + IndexNow 자동 통보
